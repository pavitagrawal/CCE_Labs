Develop an Android application that integrates multiple UI components and database functionality in a single application. The user should be able to book movie tickets using a Spinner to select movie category (such as Action, Comedy, Drama, Thriller, etc.), a Date Picker to choose the movie date with restriction to only valid future dates up to 7 days in advance, and a Time Picker to select the movie schedule with constraints allowing only valid show timings between 9:00 AM and 11:00 PM. The screen should also include Radio Buttons to select ticket type (Regular or VIP), CheckBoxes to choose additional services (Snacks, Parking, 3D Glasses), and a Toggle Button to switch between Online and Offline booking mode.

When the user clicks the Submit button, all booking details should be stored in a SQLite database table named “Bookings,” followed by displaying a Toast message confirming successful booking and navigating to a Booking Summary Screen using an Intent. The Booking Summary Screen should display all selected booking details and include a Popup Menu with functional options such as Edit Booking, Cancel Booking, and Share Booking. The Edit Booking option should navigate back to the Booking Screen with previously filled data using Intent extras, allowing the user to modify details and update the existing record in the database. The Cancel Booking option should delete the selected booking from the database, display a confirmation Toast message, and return to the Booking Screen. The Share Booking option should display booking details using a Toast message.

Additionally, the application should include a “View Bookings” button on the Booking Screen that navigates to a Booking List Screen using an Intent. This screen should retrieve all stored bookings from the SQLite database and display them using a ListView. A long press on any booking item should open a Context Menu with options to Update (which opens the Booking Screen with pre-filled data for editing) and Delete (which removes the booking from the database and refreshes the ListView). Proper Back Button navigation must be implemented such that pressing back from the Summary Screen or Booking List Screen returns to the Booking Screen, and pressing back from the Booking Screen exits the application.
